The Previous Post Saved This Page

April 10, 2022

I haven’t posted for a month despite having worked on a number of things. The reason why is I lost the ability to generate the site when I lost the markdown files for the content posts, all three of the new classes I showed off in the previous post, and pretty much every package, lib, and environment configuration that made the site build the way it did.

Losing Data Needed to Build

I lost data on the PC I had been working on the project from. It would have been all safely tucked away into the repo on github if not for some concern for uploading draft files of things I had been working on finishing but didn’t want to publish anywhere yet hadn’t made me decide to remove the site generator from the repository that hosts this page. I had planned to make a separate repo for the generator, but I didn’t. And when the file loss was complete, a good lot of work would need to be done to get the site building again.

  1. Firstly I had to reinstall pelican and the theme and all its plugins, including sass and the list goes on.

  2. I had to find old repos of old versions of the pelican site and scrap together the settings for the basic templates in this theme.

  3. I had to rewrite markdown files from every HTML post I did not have an older backup of; otherwise it would never dynamically adjust to new pages again.

  4. And at the end of all of it, if I had not posted the full pelicanconf.py file like I did in that last post, a lot of the newer automated functionality would be gone.

It took me several weeks just to want to consider working on this. And I finally did it so I can write. The next post will be about my BandCamp downloading utility.

But before moving on…

Concerns With Statically Generating Sites

I now realize that while the site may not depend on a server configuration to make it render, it does still depend somewhat on the configuration on the local machine to continue building on it.

Not just the pages that are generated, but the content files that result in the pages, and all other files and folders in the static generator have to be maintained somewhere remote where they can be pulled down again if needed.

This will no longer be an issue as I will be uploading all of it to the same place. If you want to find my potentially forgotten-about drafts in a folder somewhere in some directory, go for it.

I’ll add the folder to an upload ignore config one day if I need to.
